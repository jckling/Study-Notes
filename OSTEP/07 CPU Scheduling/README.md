## 7 CPU Scheduling

首先了解一下程序的参数和默认值

```python
parser = OptionParser()
# 随机种子
parser.add_option("-s", "--seed", default=0, help="the random seed", action="store", type="int", dest="seed")
# 作业数量，默认为 3
parser.add_option("-j", "--jobs", default=3, help="number of jobs in the system", action="store", type="int", dest="jobs")
# 作业列表
parser.add_option("-l", "--jlist", default="", help="instead of random jobs, provide a comma-separated list of run times", action="store", type="string", dest="jlist")
# 作业最长长度，默认为 10
parser.add_option("-m", "--maxlen", default=10, help="max length of job", action="store", type="int", dest="maxlen")
# 调度算法，先进先出（FIFO）、最短作业优先（SJF）、轮转（RR）
parser.add_option("-p", "--policy", default="FIFO", help="sched policy to use: SJF, FIFO, RR", action="store", type="string", dest="policy")
# 轮转调度算法的时间片长度，默认为 1
parser.add_option("-q", "--quantum", help="length of time slice for RR policy", default=1, action="store", type="int", dest="quantum")
# 计算答案
parser.add_option("-c", help="compute answers for me", action="store_true", default=False, dest="solve")

(options, args) = parser.parse_args()
```

响应时间（response time）：开始时间 - 到达时间
周转时间（turnaround time）：完成时间 - 到达时间
总等待时间（total wait time）：所有等待时间之和（非运行时间）

假设所有任作业同时到达，即到达时间为 0

例如，使用先进先出 `-p FIFO`，随机数种子 100 `-s 100`，并打印结果 `-c`
- 三个作业，长度分别为 2、5、8

运行三个长度分别为 100、200、300 的作业，采用轮转（RR）调度，时间片长度为 1；

每个作业轮流运行 1 个时间片，因此响应时间分别是 0、1、2，循环调度执行直到所有作业完成

轮转时间（=完成时间）
- 第一个作业：3*100-2 = 298
- 第二个作业：300+(2*100-1) = 499
- 第三个作业：500+100 = 600

等待时间（非运行时间）
- 第一个作业：298 - 100 = 198
- 第二个作业：499 - 200 = 299
- 第三个作业：600 - 300 = 300

## 7

随着量子长度的增加，RR 的响应时间会怎样？

- 如果工作长度小于时间片长度，则响应时间增加
- 如果工作长度大于等于时间片长度，则响应时间不变

给定 N 个工作时，最坏情况的响应时间
- 设时间片长度为 T
- 第 n 个作业的响应时间 T(n) = (n-1)T
  - n = 1,2,3,...
- 平均响应时间 T_avg = [0+T+2T+3T+...+(n-1)T]/n = (n-1)*T/2
